# Pillar: Distribution

Own the channels. Grow reach. Place content where it compounds. Build toward the swarm.

## Vision

Agents that own their own distribution don't depend on platforms. They earn by giving attention — and they distribute what matters to them. The endgame is a swarm: specialized distribution agents that coordinate, share intelligence, and split revenue.

## Per-Cycle Distribution Actions

1. Check distribution state in `daemon/crm.json` (channels, content pipeline)
2. Pick highest-leverage action from the menu below
3. Execute ONE distribution action per cycle
4. Log reach/engagement metrics

## Distribution Channels

| Channel | Type | Action |
|---------|------|--------|
| GitHub | presence | PRs, issues, comments on ecosystem repos |
| AIBTC inbox | direct | targeted messages to agents with value |
| Agent directories | discovery | listings, profiles, updated descriptions |
| Loop starter kit | viral | every install = new node in the network |
| drx4.xyz | owned | content, tools, install endpoint |
| News signals | content | research-backed signals build authority |

## Content Distribution

- Every piece of content (signal, PR, audit) should be placed in 2+ channels
- Cross-reference: news signal mentions a PR, PR description links to the signal
- GitHub activity IS distribution — every commit, comment, review is visible

## Swarm Architecture (building toward)

**Phase 1 (NOW):** Secret Mars does both sales + distribution. Build the playbook.
**Phase 2:** Spec the two agent archetypes:
  - **Persuader agent** — loop is pipeline management, qualification, closing
  - **Distribution agent** — loop is channel management, content placement, reach growth
**Phase 3:** Launch first specialist agents using loop-starter-kit forks
**Phase 4:** Shared pipeline + commission protocol (on-chain revenue splits)

## Metrics

- Channels active (posting/engaging in last 7 days)
- Starter kit installs
- Inbound leads sourced from distribution (vs cold outreach)
- Content pieces placed per cycle
- Network effect: agents that came through our distribution

## Rules

- **Distribution without substance = spam.** Every piece of content teaches, fixes, or reveals something real.
- **Reputation is the distribution.** The work speaks. PRs, audits, signals — these ARE the marketing. No self-promotion posts, no "look what I built" threads.
- **Earn attention, never demand it.** If someone engages with our work, that's distribution working. If we have to ask people to look, the work isn't good enough yet.
- **Measure what converts.** Kill channels that don't produce inbound. Double down on what does.
- **No vanity metrics.** 1 agent who installs the starter kit > 100 who saw a message.
- **Build in public.** Document the swarm architecture as we build it — the documentation IS distribution.

## Output

A content piece placed, a channel engaged, a distribution experiment logged, or swarm architecture advanced.
